## ISOとマネジメントシステム規格

ISOはInternational Organization for Standardizationの略であり、国際標準化機構を指します。本部はスイスのジュネーブにあります。マネジメントシステム規格とは、組織を運営するための仕組みを認証の対象とする規格です。認証は第三者機関によって審査・登録され、その後も継続的な運用を確認するため定期審査を受ける必要があります。

## 主要なマネジメントシステム規格

| 規格 | 対象 | 通称 |
| --- | --- | --- |
| ISO 9001 | 品質マネジメント | QMS |
| ISO 14001 | 環境マネジメント | EMS |
| ISO 27001 | 情報セキュリティマネジメント | ISMS |
| ISO 22301 | 事業継続マネジメント | BCMS |
| ISO 45001 | 労働安全衛生マネジメント | OHSMS |

## 各規格の概要

### ISO 9001（品質マネジメントシステム）

ISO 9001は品質マネジメントシステム（QMS）の規格であり、顧客に対して高品質な製品やサービスを提供する仕組みを定めています。この規格ではPDCAサイクルを基本としており、継続的な改善が組織に要求されます。

### ISO 14001（環境マネジメントシステム）

ISO 14001は環境マネジメントシステム（EMS）の規格で、組織の活動に伴う環境負荷を低減するための仕組みを提供します。廃棄物の削減やエネルギー効率の向上、汚染防止などが管理の対象となります。

### ISO 27001（情報セキュリティマネジメントシステム・ISMS）

ISO 27001は情報セキュリティマネジメントシステム（ISMS）の国際規格であり、情報資産の機密性・完全性・可用性（CIA）を保護する仕組みを求めます。リスクアセスメントから始まり、対策の実施、監査、改善へと続くサイクルで運用されます。技術面の詳細は[暗号化の基礎](/blog/encryption-basics)や[認証と認可の違い](/blog/auth-vs-authz)で解説しています。

### ISO 22301（事業継続マネジメント）

ISO 22301は事業継続マネジメントシステム（BCMS）の規格であり、災害時に事業を継続する仕組みを定めています。これは[BCP（事業継続計画）](/blog/bcp-toha)を国際規格化したものです。

## その他重要な ISO 規格

その他の重要なISO規格として、ISO/IEC 25010はソフトウェア品質特性を定義します。ISO/IEC 20000はITサービスマネジメントの国際規格であり、ITILをベースにしています。ISO/IEC 12207はソフトウェアライフサイクルプロセスを定めます。

## ITパスポート試験での出題ポイント

ITパスポート試験では、各ISO番号とその対象の対応を問う問題が最頻出です。ISMSの中核であるCIA（機密性・完全性・可用性）の理解も必須です。また、マネジメントシステムの基本としてPDCAサイクルとの関係も押さえておきましょう。

### 過去問の典型パターン

- 「情報セキュリティの国際規格はどれか」型 → ISO 27001
- 「ISO 14001 が対象とするものはどれか」型 → 環境

## 学習のコツ

学習のコツとして、ISO番号と対象の対応は「9は品質、14は環境、27は情報セキュリティ」と語呂で覚えるとよいでしょう。ISMSのCIAは機密性・完全性・可用性の3つを必ず言えるようにしておきます。また、マネジメントシステムの基本はPDCAサイクルであることを意識しておきましょう。
