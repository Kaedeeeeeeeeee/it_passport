## DHCPとは

DHCP（Dynamic Host Configuration Protocol）は、ネットワークに接続した端末に対してIPアドレスなどの設定情報を自動的に割り当てるプロトコルです。手動で設定する手間を省き、入力ミスやアドレスの重複を防止できます。サーバ側はUDP 67番ポート、クライアント側はUDP 68番ポートを使用して通信します。

## なぜDHCPが必要か

IPアドレスを手動で設定すると、設定ミス（重複や誤入力）が発生しやすく、多数の端末を管理する負担が大きくなります。特にノートPCやスマートフォンなど移動する端末では、ネットワークを変えるたびに再設定が必要となり現実的ではありません。DHCPを利用すれば、端末はネットワークに接続するだけで自動的に通信可能な状態になります。

## DHCPで割り当てられる情報

DHCPサーバはIPアドレス、サブネットマスク、デフォルトゲートウェイ、DNSサーバのアドレスをクライアントに配布します。また、割り当てたIPアドレスを使い続けられるリース期間も合わせて通知されます。

## DHCPの通信フロー（DORA）

DHCPによるアドレス割り当ては4つのステップで行われます。最初にクライアントがネットワーク全体に「DHCPサーバはいませんか」とブロードキャストで問い合わせます（Discover）。次にサーバが「このIPアドレスはいかがですか」と候補を返します（Offer）。クライアントが「そのIPアドレスをください」と要求し（Request）、サーバが「OK、割り当てました」と確認することで完了します（Acknowledge）。この一連の流れをDORAと覚えましょう。

## リース期間

割り当てられたIPアドレスには有効期限（リース期間）が設定されています。クライアントは期限の半分を過ぎた時点で更新を試み、更新が成功すれば継続利用できます。期限が切れた場合はアドレスを解放し、再度取得する必要があります。

## 固定IPとの使い分け

| 用途 | 推奨 |
| --- | --- |
| 一般 PC・スマホ | DHCP（自動） |
| サーバ・プリンタ | 固定 IP（変わらないアドレスが必要） |
| ネットワーク機器 | 固定 IP |

## DHCPのセキュリティ課題

DHCPの仕組みを悪用すると、不正なDHCPサーバを設置して偽の情報を配布するDHCPスプーフィング攻撃が可能です。対策として、スイッチで正規のDHCPサーバのみを許可するDHCPスヌーピング機能を有効にします。
