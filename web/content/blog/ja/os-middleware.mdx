## ソフトウェアの階層

```
┌──────────────────┐
│   アプリケーション   │  例：Word、Chrome
├──────────────────┤
│   ミドルウェア     │  例：DBMS、Web サーバ
├──────────────────┤
│   オペレーティングシステム │  例：Windows、Linux
├──────────────────┤
│      ハードウェア    │  例：CPU、メモリ
└──────────────────┘
```

## OS（Operating System）の役割

### プロセス管理

OSは複数のプロセス（実行中のプログラム）をCPUに割り当てる管理を行います。これにより並行処理（マルチタスク）を実現し、複数のアプリを同時に動かせます。スケジューリング方式としてはラウンドロビンや優先度方式などが用いられます。

### メモリ管理

各プロセスに対してメモリを割り当てるのがOSの役割です。仮想記憶の仕組みを使うと、物理メモリ以上の容量をあたかも使えるように見せかけられます。ページングやスワッピングといった技術で効率的に管理します。

### ファイル管理

ファイルやディレクトリの作成・読み書き、権限管理はOSが担当します。代表的なファイルシステムとして、WindowsではNTFS、macOSではAPFS、Linuxではext4が使われます。

### デバイス管理

入出力装置の制御はOSが行い、デバイスドライバを介してハードウェアと通信します。キーボードやマウス、プリンタなど、多様な機器を統一的に扱えるようにします。

### セキュリティ

ユーザ認証やアクセス権限の管理もOSの重要な仕事です。複数のユーザが使う環境では、他者のファイルを勝手に見られないように制御します。

## ミドルウェアとは

OSとアプリケーションの間を仲介するソフトウェアがミドルウェアです。共通機能をアプリに提供することで、アプリ開発の効率を高めます。代表例として、データベース管理システム（MySQL、PostgreSQL）やWebサーバ（Apache、nginx）があります。また、アプリケーションサーバ（Tomcat）やメッセージング（Kafka）もミドルウェアに分類されます。
