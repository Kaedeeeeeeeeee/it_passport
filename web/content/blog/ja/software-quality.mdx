## ISO/IEC 25010とは

ソフトウェアの品質特性を国際的に統一して定義した規格がISO/IEC 25010です。旧称はISO/IEC 9126で、2011年に25010へ改定されました。この規格ではソフトウェア品質を8つの主特性に分類しています。ITパスポート試験では、この8特性の名称と内容を問う問題が頻出します。

## 8 品質特性一覧

| 特性 | 内容 | 例 |
| --- | --- | --- |
| 機能適合性 | 機能要件をどれだけ満たすか | 仕様通りの動作 |
| 性能効率性 | 応答時間・スループット | 1 秒以内に応答 |
| 互換性 | 他システムとの共存・置換 | API 互換性 |
| 使用性（ユーザビリティ） | 操作のしやすさ | 学習しやすい UI |
| 信頼性 | 故障せず動き続けるか | 99.9% 稼働率 |
| セキュリティ | 不正アクセスを防げるか | 暗号化、認証 |
| 保守性 | 修正・拡張のしやすさ | コードの可読性 |
| 移植性 | 別環境への移しやすさ | OS をまたいで動く |

## 各特性の詳細

### 機能適合性

機能適合性は、機能完全性、機能正確性、機能適切性という三つの副特性から構成されます。簡単に言えば「仕様書に書かれた通りの機能をシステムが提供しているか」を評価する特性です。試験では「要件を満たす機能が実装されているか」という問題で問われます。

### 性能効率性

性能効率性は、時間効率性、資源効率性、容量に関する特性です。具体的には応答時間やCPU・メモリ使用量など、システムの処理効率を評価します。例えば「1秒以内に画面が切り替わるか」といった問題で問われることが多いです。

### 信頼性

信頼性は、可用性、成熟性、フォールトトレランス、回復性の副特性を持ちます。一言で表すと「壊れにくく、壊れても早期に回復する性質」です。試験では「稼働率99.9%」といった数値が信頼性の例としてよく出題されます。

### セキュリティ

セキュリティは、機密性、完全性、否認防止性、責任追跡性、真正性を包含する特性です。不正アクセスを防ぎ、データを保護する能力を評価します。技術面の理解を深めたい方は、[暗号化の基礎](/blog/encryption-basics)や[認証と認可の違い](/blog/auth-vs-authz)の記事も参照しましょう。

### 保守性

保守性は、モジュール性、再利用性、解析性、修正性、試験性という副特性で構成されます。「コードの可読性が高く、後から修正や機能追加がしやすいか」を評価する特性です。リファクタリングやテストの容易さが問われる問題で出ます。

### 移植性

移植性は、適応性、設置性、置換性の副特性を持ちます。クラウドやOSをまたいでソフトウェアを動作させる際に、どれだけスムーズに移行できるかを評価します。試験では「異なる環境でも同じように動くか」という観点で問われます。

## ITパスポート試験での出題ポイント

試験では、8品質特性の名称と内容の対応問題が頻出です。各特性の具体的な例を選ばせる問題もよく出ます。また、ISO/IEC 25010自体がソフトウェア品質特性を定義した規格であることもしっかり覚えておきましょう。

### 過去問の典型パターン

- 「99.9% の稼働率は何の品質特性に該当するか」型 → 信頼性
- 「画面操作のしやすさは何に該当するか」型 → 使用性

## 学習のコツ

8つの特性を「機・性・互・使・信・セ・保・移」という語呂で覚えると記憶に定着しやすいです。各特性の代表的な数値や指標を一つだけ関連付けて覚えましょう。セキュリティについては、CIA（機密性・完全性・可用性）に加えて否認防止性と真正性も押さえる必要があります。

## まとめ

8品質特性の名称と内容、そしてISO/IEC 25010の存在を確実に押さえれば、関連問題で得点を稼げます。マネジメント系をさらに網羅的に演習するなら[マネジメント系まとめ](/category/management)、本番形式の問題に挑戦するなら[模擬試験](/exam)を活用してください。
