## データ構造とは

データ構造は、データを効率的に格納・操作するための形式です。用途に応じて適切な構造を使い分けることで、処理速度やメモリ効率が大きく変わります。

## 線形構造

### 配列（Array）

配列は、同じ型のデータを連続したメモリ領域に格納します。添字を用いてアクセスするため、アクセス時間は**O(1)** と高速です。一方、要素の挿入や削除には、後続の要素をずらす必要があるため O(n) の時間がかかり、苦手とされています。

### 連結リスト（Linked List）

連結リストでは、各要素が「データ」と「次の要素へのポインタ」で連結されています。挿入・削除は位置がわかれば **O(1)** で可能ですが、添字によるアクセスには O(n) かかります。

### スタック（Stack・LIFO）

スタックは **LIFO（Last In First Out）**、つまり後入れ先出しのデータ構造です。主な操作は push（追加）と pop（取出）です。実際の例としては、関数呼び出し履歴、Undo 機能、ブラウザの戻るボタンなどが挙げられます。

### キュー（Queue・FIFO）

キューは **FIFO（First In First Out）**、つまり先入れ先出しのデータ構造です。操作は enqueue（追加）と dequeue（取出）です。プリンタの印刷待ちやメッセージキューなどで利用されます。

### 双方向キュー（デック・Deque）

両端から追加と取出が可能なキューです。スタックとキューの両方の特性を持ち合わせています。

## 階層構造

### 木構造（Tree）

木構造は、親子関係を持つ階層構造です。ノード（節点）と枝（エッジ）で構成されます。用語として、根（root）、葉（leaf）、深さ、高さがあります。

### 二分木（Binary Tree）

二分木は、各ノードが最大 2 つの子を持つ木です。特に二分探索木では、左 < 親 < 右の順序で配置されるため、探索が O(log n) で行えます。

## その他

### ハッシュテーブル

ハッシュテーブルは、キーをハッシュ関数で添字に変換して値を格納します。平均 O(1) で検索・挿入・削除が可能ですが、衝突時の処理（チェイン法やオープンアドレス法）が必要です。

### グラフ

グラフは、ノードとエッジで関係を表現します。ネットワーク図、SNS の友達関係、地図の経路などに利用されます。

## ITパスポート試験での出題ポイント

スタック（LIFO）とキュー（FIFO）の違いは頻出です。また、配列と連結リストのアクセス・挿入にかかる計算量、木構造の用語（根・葉・深さ）も押さえておきましょう。

### 過去問の典型パターン

- 「後入れ先出しのデータ構造はどれか」型 → スタック
- 「プリンタの印刷待ち行列に適したデータ構造はどれか」型 → キュー

## 学習のコツ

LIFO はスタック（皿の積み重ね）、FIFO はキュー（行列）と例えると覚えやすいです。「配列 = 高速アクセス、リスト = 高速挿入」と特性で対比しましょう。木の用語（根・葉）は一度図解すると理解が深まります。
